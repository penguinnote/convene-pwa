# 디자인 기록 (DESIGN.md)

Convene의 디자인 의사결정과 디자인 시스템을 정리한 기록이다. 무엇을 시도했고 왜 최종안을 택했는지를 담는다. 사례와 톤은 첫 배포 인스턴스인 로뎀나무교회 청년대학부 여름캠프를 기준으로 한다. (행사별로 달라지는 색·표어는 `src/config/instance.js`에 분리돼 있어, 아래 팔레트도 그중 한 인스턴스의 값이다.)

---

## 1. 디자인 목표와 원칙

- **모던 미니멀, 화이트 베이스**: 정보(공지·일정·말씀)가 우선이므로 장식을 절제하고 여백과 가독성을 중심에 둔다.
- **브랜드 일관성**: 참가자는 행사 포스터를 먼저 접한 뒤 앱을 설치한다. 앱이 포스터와 같은 색·무드를 공유해야 하나의 캠페인으로 인식된다.
- **모바일 우선**: 80~100명이 폰으로 쓰므로 모든 결정의 기준은 좁은 화면과 작은 글씨의 가독성이다.

---

## 2. 색상 결정 과정

초기엔 차분한 바질 그린을 포인트로 썼으나 실제 브랜드 색은 아니었다. 캠프 포스터의 정체성은 **맑은 하늘빛·아쿠아 파스텔 + 흰빛 + 금빛 반짝임**의 서정적 수채화 톤이었고, 브랜드 일관성을 위해 전체 색을 포스터 계열로 전환했다. 파스텔을 그대로 쓰면 대비가 부족해 포인트는 한 단계 진한 톤을 썼다.

| 용도               | 값        |
| ------------------ | --------- |
| 강조 / 활성 / 링크 | `#3F7D99` |
| 제목 및 공지 제목  | `#2F5E72` |
| 카드 배경          | `#F2F8FA` |
| 테두리             | `#D4E6EC` |
| 본문               | `#21343C` |
| 보조 텍스트        | `#52707D` |
| 흐린 텍스트        | `#8AA6B3` |

팔레트는 인스턴스 설정에 두고 Tailwind와 매니페스트가 함께 참조한다. 새 행사는 이 값만 바꾸면 앱 전체 색이 일괄 전환된다.

## 3. 타이포그래피

영문 표어 "Where are you?"를 세리프 이탤릭으로 두기로 하면서 어울리는 한글 폰트를 탐색했다. 산세리프(Pretendard 등)는 세리프 표어와 결이 안 맞았고, 나눔명조는 붓터치가 강해 올드했으며, Hahmlet은 우아하나 각졌다. 최종은 **Gowun Batang** — 부드럽고 둥근 바탕체로 포스터의 포근한 무드, 영문 세리프와 가장 잘 맞았다.

- 전체 폰트를 Gowun Batang으로 통일한다.
- 제목은 가는 무게(400)로 두어 포스터의 가늘고 우아한 인상을 따른다.
- 제목·공지 제목 색은 `#2F5E72`로 통일하고, 영문 표어는 세리프 이탤릭으로 포인트를 살린다.
- 한글 줄바꿈은 `word-break: keep-all`을 쓰되, 띄어쓰기 없는 긴 문자열이 넘칠 때만 `overflow-wrap`으로 강제 줄바꿈해 가로 오버플로우를 막았다.

## 4. 홈 구성

홈은 헤더 아래에 **최신 공지 1건, 라이브 현재 순서, 자료실**을 한 화면에 담는다.

초기 홈엔 창세기 3:9 blockquote "주제 말씀" 블록을 뒀으나 헤더가 이미 같은 본문을 담아 정보가 중복이었다. 블록을 제거해 공지 섹션을 히어로 바로 아래로 끌어올렸고, 앱을 열면 가장 중요한 공지가 첫 화면에 바로 보이게 했다. 공지는 홈에서 최신 1건만 노출하고 상대시간(방금 전 / N분 전)을 표시하며, 미리보기는 입력한 줄바꿈을 그대로 반영한다. 고정(자료실) 공지는 위계를 낮춰 분리한다.

- **하단 탭 순서**: 홈 / 일정 / 말씀 / 사진 / 정보. 사진은 라우트가 아니라 외부 Google Photos 앨범으로 연결하고, 방배정·식단표·플레이리스트·레크레이션·참여자는 정보 탭 하위로 묶었다.
- **수채화 헤더**: 상단에 CSS 그라데이션(라디얼 + 리니어)으로 수채화 질감을 표현하고 금빛 반짝임을 절제해 배치한다. "진하되 밝게"를 기준으로 명도·채도를 맞췄다.

## 5. 라이브 "현재 순서" 카드 — 세 번의 설계

홈 중앙에 "지금 진행 중"을 보여주는 카드다. 설계는 세 단계를 거쳤다.

① 처음엔 기기 시계로 현재 순서를 자동 계산하고 D-Day 카운트다운까지 붙였다. ② 실제 캠프는 집회가 길어지거나 식사가 당겨져 순서가 10~40분씩 밀리고 당겨진다 — 시계만으로는 "지금"을 자주 틀리게 짚는다. D-Day를 걷어내고 관리자가 포인터를 손으로 넘기는 수동 방식으로 바꿔 봤다. ③ 하지만 매 순서 버튼을 눌러 넘기는 건 놓치기 쉽고 마찰이 컸다. 결국 **일정표 기준 시각 자동 계산으로 되돌리되 변동 안내 메모는 관리자 몫으로 남기는** 절충으로 정리했다. 시각으로 큰 흐름을 맞추고, 밀림·당김 같은 예외만 사람이 한 줄 메모("점심 30분 지연")로 덧댄다. 자동의 손 안 가는 이점과 수동의 정확성을 나눠 가진 선택이다.

자동 계산은 순수 함수 `getAutoLive(now)`가 맡는다. `LIVE_ANCHOR`(캠프 첫날) 기준 경과일을 일정 길이로 나눈 나머지로 그날 일차를 정하고, 현재 시:분으로 진행 중/다음 순서를 뽑는다. 자정~아침 첫 순서(07:00) 사이엔 "취침"으로 조용히 채우고 다음 일정만 보여 준다.

카드는 현재 순서(시각·제목·장소)를 강조하고 아래에 다음 순서를 흐린 톤으로 얹는다. 각 일정엔 성격에 맞는 이동 링크(강의→말씀 본문, 프로그램→자료실, 식사→식단표, 묵상→말씀 탭, 찬양·쉬는 시간→플레이리스트, 레크레이션→조 편성)를 한 칩으로 달아 "지금 뭘 보면 되는지"를 한 번의 탭으로 잇는다. 이동 대상은 각 일정의 `link` 타입으로 정해진다. 톤은 공지 highlight 카드(2px)보다 약한 1px 테두리로 두어, 공지가 홈의 주인공 자리를 지키게 했다.

## 6. 말씀 탭 — 목록에서 본문, 3역본, 가독성

말씀 본문은 처음엔 아코디언으로 제자리에서 펼쳤으나 긴 장에선 목록이 밀려 탐색이 불편했다. **목록 → 개별 본문 페이지** 구조로 바꿔, 강의를 고르면 전용 페이지에서 읽게 했다. 각 항목에 안정적 id를 부여해 라이브 카드 링크와 상세 라우트가 같은 키로 연결되게 했다.

본문은 **개역개정·NIV·새번역 3역본 토글**을 지원한다(원문 대조·묵상용). 가독성은 세 가지로 다졌다. 절과 절 사이를 한 줄 띄워 성경 조판처럼 읽히게 하고(비연속 구간은 더 큰 간격), `break-keep`에 `overflow-wrap`을 더해 모바일 좁은 폭에서 단어가 잘리거나 넘치지 않게 했으며, 4단계 글씨 크기 조절(기본값을 아래에서 둘째로 두어 한 단계 축소·두 단계 확대)을 붙이고 선택값을 `localStorage`에 저장해 재진입·다른 말씀에서도 유지되게 했다. 뒤로가기는 본문 → 말씀 탭 → 홈의 2단계로 복귀한다.

## 7. 레크레이션 조 편성 — 게임형 팀 배정

무작위로 "너는 3조"라 통보하는 대신, 취향 질문에 답하며 자기 팀을 발견하게 만든 아이스브레이킹 도구다. 3라운드(음식·초능력·성향) × 4문제 2지선다에 답하면 16개 구역(A1~D4) 중 하나로 배정되고, 결과 화면이 "B구역 1번 표지로" 이동을 안내해 온라인 배정과 오프라인 동선을 잇는다.

- **결정 방식**: 인원 균형(카운터형)보다 "같은 취향 = 같은 팀"의 재미를 택했다. 답 4개를 4비트로 보아 앞 2문제로 열, 뒤 2문제로 행을 정하는 전단사 매핑이라 서버 없이 결정되고, 코드에서 답을 되복원할 수 있어 결과에 내가 고른 선택지도 함께 보여 준다.
- **공유되는 팀 표**: 배정은 참가자 프로필 문서(`users.teams`)에 저장되고, 전체를 16칸 표로 실시간 집계해 누구나 조 편성 전체를 확인한다. 로그인 때 받은 이름·목장이 자동으로 칸에 채워져 따로 입력하지 않는다. 칸 크기는 모바일 2열(세로로만 늘어남)·데스크톱 4열(넓게)로 다르게 두었다.
- **진행 흐름**: 진입 시 바로 설문을 띄우지 않고 시작 페이지를 두었고, 배정은 확정(재선택 없음)이며, 관리자가 열어야 참여할 수 있다. 관리자는 라운드별 시작·종료·재시작과 전체 재시작, 개별 인원 이동·삭제를 제어한다.

## 8. 참여자 커뮤니티 & 프로필

익명 로그인 기반 온보딩에서 이름·목장·프로필 사진을 설정한다. 사진은 원형 크롭 UI로 얼굴을 맞춰 올리고 이후 화면에서 원형 아바타로 쓴다.

이 로그인 정보를 커뮤니티로 확장했다. **참여자 탭**은 전체 참가자의 이름·목장·사진을 실시간으로 모아 보여 주고, 프로필 카드엔 "팔로워·팔로잉"을 전 참가자 수로 표시했다. 실제 팔로우/언팔로우 관계 그래프를 만드는 대신(자동 상호 팔로우는 결국 "전원 목록 + 인원수"와 같으므로) 숫자 연출만 얹어, "이 캠프의 모두가 서로 친구이자 동역자"라는 메시지를 최소 복잡도로 전했다.

## 9. 관리자 콘솔

관리자는 이메일 로그인(`isAdmin = !!user.email`) 전용으로, 공지 목록·블록 에디터, 라이브 변동 메모, 조 편성 진행 관리(라운드 시작·종료·재시작, 인원 이동·삭제), 참여자 조회·수정·삭제를 한 화면 안 뷰 전환으로 오간다. Admin은 모바일 셸과 데스크톱 프레임에서 같은 컴포넌트로 렌더돼 폰·데스크톱 모두에서 쓰인다.

**접근 통제**: 앱이 전원 익명 로그인시키는 탓에 "로그인 여부"만으로는 관리자를 가릴 수 없어, 익명 사용자에게 관리자 화면이 노출되는 문제가 있었다. UI 게이트를 이메일 보유 기준으로 바꾸고 Firestore 규칙에서 공지·설정 쓰기를 이메일 관리자 조건으로 제한해, 화면과 데이터 양쪽을 막았다.

## 10. 앱 아이콘 · 스플래시

- **아이콘**: 초기 그린 테마에서 포스터의 블루 테마로 재디자인했다. 흰 배경에 밝은 청색 `#67BDDB` 반짝임 심볼 하나만 두어 작은 크기에서도 또렷하고, 앱 전체 색과 홈 화면 아이콘의 인상을 일치시켰다.
- **스플래시**: 콜드 스타트마다 흰 배경에서 아이콘 → 로딩 막대 → 홈으로 이어진다. 1단계는 아이콘만 보여 브랜드를 각인하고, 2단계에서 아이콘이 작아지며 위로 올라가고 그 아래 막대가 브랜드 그라데이션으로 채워진 뒤 페이드아웃으로 홈에 넘어간다. 타이밍은 상수로 분리해 차분한 인상을 주도록 넉넉히 맞췄다.

## 11. 인앱 토스트

앱을 쓰는 동안엔 시스템 알림이 뜨지 않는 플랫폼 동작이 있어, 포그라운드에서 새 공지가 등록되면 화면 상단에 인앱 토스트로 알린다. 흰 카드에 1px 테두리와 부드러운 그림자, 좌측 원형 블루 종 아이콘으로 공지임을 알린다. 위에서 슬라이드 인해 4초 뒤 사라지고, 탭하면 상세로 이동, 위로 스와이프하면 닫힌다(살짝 떼면 탭, 충분히 끌면 닫기).

## 12. 공지 첨부 디자인

목록에선 일관성을, 상세에선 원본 충실도를 우선했다.

- 홈 카드는 우측에 정사각 썸네일을 두어 길이가 다른 공지도 같은 리듬으로 정렬하고, 이미지가 없으면 썸네일을 생략해 본문이 폭 전체를 쓴다. 파일 첨부는 첨부 칩(파일명 말줄임)으로 한 줄을 유지한다.
- 자료실(고정 공지) 카드는 상단 공백과 업로드 시간을 없애고 제목이 바로 눈에 닿게 했다.
- 상세 페이지는 이미지를 원본 비율 그대로(가로 꽉, 높이 자동) 보여 크롭하지 않는다. 파일은 좌측 정사각 PDF 썸네일 + 우측 파일명·다운로드 안내로 구성하고 탭 시 새 탭으로 연다.

## 13. 디자인과 기술의 교차점

### iOS PWA 상태바
- iOS는 `theme-color` 메타가 있으면 그 값으로 상태바를 단색으로 칠한다. 헤더의 수채화 그라데이션을 상태바까지 자연스럽게 잇기 위해 이 동작을 고려했다.
- `apple-mobile-web-app-status-bar-style`은 홈 화면 추가 시점에 한 번만 읽혀, 값을 바꾸면 재설치해야 적용된다.
- Android는 상태바가 OS가 칠하는 단색이라 그라데이션이 불가능하며, 매니페스트 `theme_color`를 그라데이션 상단 색과 맞추는 것이 최선이다.

### 하단 탭 고정
- `position: fixed`는 iOS PWA에서 `viewport-fit=cover`와 동적 뷰포트가 맞물려 위치가 불안정했다. 최상위를 `100dvh` 세로 flex로 두고 본문만 내부 스크롤하게 전환, 하단 탭은 flex 자식으로 맨 아래에 배치해 해결했다. 하단 안전영역은 `env(safe-area-inset-bottom)` 패딩으로 유지한다.

### 넓은 화면 프레임
- 넓은 화면에서 모바일 UI를 유지하는 중간 단계로, 앱을 `max-w-md` 중앙 컬럼으로 두고 좌우 여백을 `#root`의 은은한 아쿠아 그라데이션으로 채우고 `md:shadow-xl`로 떠 보이게 했다. 이 프레임은 폰보다 살짝 넓은 구간(약 448~767px)에서 드러나고, 그 위(≥768px)부턴 데스크톱 셸이 대신한다.

### 데스크톱 레이아웃 (넓은 화면 전용 셸 + 홈 대시보드)
- 폰과 넓은 화면은 정보 밀도가 달라, 단일 열을 늘리는 대신 ≥768px엔 별도 데스크톱 셸을 뒀다. 전환은 `useIsDesktop`(matchMedia) 훅으로 하고, 그 미만(폰)은 기존 모바일 셸(하단 탭·`max-w-md` 중앙 열)을 픽셀 그대로 유지한다. 모바일 컴포넌트는 건드리지 않고 데스크톱 전용 컴포넌트를 별도로 두어 회귀 위험을 없앴다.
- **공통 셸**: 전체 폭 수채화 히어로 + 가로 탭 내비(홈·일정·말씀·사진, 활성 탭 밑줄). "사진"은 외부 앨범을 새 탭으로 열고, "정보"는 데스크톱 탭에 두지 않는다. 하단 탭은 숨긴다.
- **홈 대시보드**: 라이브 와이드 카드(좌 현재·우 다음+링크 칩), 그 아래 공지 최신 1건과 오늘 일정 미리보기 2열, 자료실 카드로 구성한다. 오늘 일정 카드는 밑단을 자료실과 맞추고, 남은 순서가 적으면 이전 순서로 채운다. 오른쪽 정보 레일(lg+)엔 프로필 카드(팔로우 수 포함)와 INFO 바로가기(참여자·방배정·플레이리스트·식단표·레크레이션)를 두고 좁은 폭에선 접힌다. 데이터·라이브·링크 로직은 `useHomeData` 훅으로 모바일 홈과 공유하고 배치만 재구성했다.
- **콘텐츠 페이지**(데스크톱 전용 컴포넌트): 일정은 가운데 정렬 넓은 타임라인("지금" 강조), 말씀은 좌 목록 + 우 본문 2단 마스터-디테일(라우트 이동 없이 선택 전환), 방배정은 방 유형별 섹션 카드 그리드, 조 편성·참여자·식단·플레이리스트·공지·관리자·온보딩도 넓은 화면용으로 배치한다(기능 로직 변경 없이 폭·정렬만).
- **정보 탭 이동**: 모바일 "정보" 탭 콘텐츠는 데스크톱에선 홈 오른쪽 레일로 옮겼다. 그래서 데스크톱 내비엔 "정보" 탭이 없고 `/info`(프로필 편집) 진입점은 레일의 "프로필 수정" 하나로만 남긴다. 하위 라우트는 그대로 유지한다.

### 뒤로가기
- 어느 경로로 들어왔든 항상 같은 부모 화면으로 복귀하게 해 위치를 예측 가능하게 했다. 자식 화면은 홈으로, 상세는 목록으로 돌아간다. 홈에선 뒤로가기로 앱이 종료되지 않게 머무르고, 방배정에선 검색만 해제한다.

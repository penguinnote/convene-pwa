# 디자인 기록 (DESIGN.md)

이 문서는 Convene의 디자인 의사결정 과정과 디자인 시스템을 정리한 기록이다. 사례와 톤은 첫 배포 인스턴스인 로뎀 캠프를 기준으로 든다. 무엇을 시도했고 왜 최종안을 택했는지를 담는다. (행사별로 달라지는 색·표어는 `src/config/instance.js`에 분리돼 있어, 아래 팔레트도 그중 한 인스턴스의 값이다.)

---

## 1. 디자인 목표와 원칙

- **모던 미니멀, 화이트 베이스**: 정보(공지·일정·말씀)가 우선이므로 장식을 절제하고 여백과 가독성을 중심에 둔다.
- **브랜드 일관성**: 참가자는 행사 포스터를 먼저 접한 뒤 앱을 설치한다. 앱이 포스터와 같은 색·무드를 공유해야 하나의 캠페인으로 인식된다.
- **모바일 우선**: 80~100명이 폰으로 쓰므로 모든 결정의 기준은 좁은 화면과 작은 글씨의 가독성이다.

---

## 2. 색상 결정 과정

초기에는 차분한 바질 그린을 포인트로 썼으나 실제 브랜드를 반영한 색은 아니었다. 캠프 포스터를 확인하니 정체성은 **맑은 하늘빛, 아쿠아 파스텔 + 흰 빛 + 금빛 반짝임**의 서정적 수채화 톤이었고, 브랜드 일관성을 위해 전체 색을 포스터 계열로 전환했다. 파스텔을 그대로 쓰면 대비가 부족해 포인트는 한 단계 진한 톤을 사용했다.

| 용도 | 값 |
|---|---|
| 강조 / 활성 / 링크 | `#3F7D99` |
| 제목 및 공지 제목 | `#2F5E72` |
| 카드 배경 | `#F2F8FA` |
| 테두리 | `#D4E6EC` |
| 본문 | `#21343C` |
| 보조 텍스트 | `#52707D` |
| 흐린 텍스트 | `#8AA6B3` |

팔레트는 인스턴스 설정(`instance.js`)에 두고 Tailwind와 매니페스트가 함께 참조한다. 새 행사는 이 값만 바꾸면 앱 전체 색이 일괄 전환된다.

## 3. 타이포그래피

영문 표어 "Where are you?"를 세리프 이탤릭으로 두기로 하면서 어울리는 한글 폰트를 탐색했다. Pretendard 등 산세리프는 세리프 표어와 결이 안 맞았고, 나눔명조는 붓터치가 강해 올드했으며, Hahmlet은 우아하나 다소 각졌다. 최종은 **Gowun Batang** — 부드럽고 둥근 바탕체로 포스터의 포근한 무드, 영문 세리프와 가장 잘 맞았다.

- 전체 폰트를 **Gowun Batang**으로 통일한다.
- 제목은 가는 무게(400)로 두어 포스터의 가늘고 우아한 인상을 따른다.
- 제목·공지 제목 색은 `#2F5E72`로 통일하고, 영문 표어는 세리프 이탤릭으로 포인트를 살린다.
- 한글 줄바꿈은 `word-break: keep-all`을 쓰되, 띄어쓰기 없는 긴 문자열이 가로로 넘치지 않도록 `overflow-wrap`으로 넘칠 때만 강제 줄바꿈하고 콘텐츠 영역의 가로 오버플로우를 막았다.

## 4. 홈 구성

홈은 히어로(표어) 아래에 **공지 최신 1건 · 라이브 현재 순서 · 자료실**을 한 화면에 담는다.

초기 홈은 히어로 아래에 창세기 3:9 blockquote "주제 말씀" 블록을 뒀으나, 히어로가 이미 같은 본문을 담아 정보가 중복이었다. 블록을 제거해 공지 섹션을 히어로 바로 아래로 끌어올렸고, 앱을 열면 가장 중요한 공지가 첫 화면에 바로 보이게 했다. 공지는 홈에서 **최신 1건만** 노출하고 우측 상단에 상대시간(just now / N분 전)을 표시하며, 클릭 시 상세로 "전체 보기"로 목록으로 이동한다. 고정(자료실) 공지는 화면 맨 아래로 내려 위계를 분리했다.

- **하단 탭 순서**: 홈 / 일정 / 말씀 / 사진 / 정보. 사진은 라우트가 아니라 외부 Google Photos 공유 앨범으로 연결하고, 방배정·식단표·플레이리스트는 정보 탭 하위로 묶었다.
- **수채화 헤더**: 홈과 각 페이지 상단에 CSS 그라데이션(라디얼 + 리니어)으로 수채화 질감을 표현하고 금빛 반짝임을 절제해 배치한다. "진하되 밝게"를 기준으로 명도·채도를 맞췄다.

## 5. 라이브 "현재 순서" 카드 — 세 번의 설계

홈 중앙에 "지금 진행 중"을 보여주는 카드다. 설계는 세 단계를 거쳤다.

① 처음엔 기기 시계로 현재 순서를 자동 계산하고 D-Day 카운트다운까지 붙였다. ② 그런데 실제 캠프는 집회가 길어지거나 식사가 당겨져 순서가 10~40분씩 밀리고 당겨진다 — 시계만으로는 "지금"을 자주 틀리게 짚는다. 그래서 D-Day를 걷어내고 **관리자가 포인터를 손으로 넘기는 수동 방식**으로 바꿔 봤다. ③ 하지만 사람이 매 순서 버튼을 눌러 넘기는 건 놓치기 쉽고 마찰이 컸다. 결국 **일정표 기준 시각 자동 계산으로 되돌리되, 변동 안내 메모는 관리자 몫으로 남기는** 절충으로 정리했다. 시각으로 큰 흐름을 맞추고, 밀림·당김 같은 예외만 사람이 한 줄 메모("점심 30분 지연")로 덧댄다. 자동의 손 안 가는 이점과 수동의 정확성을 나눠 가진 선택이다.

자동 계산은 순수 함수 `getAutoLive(now)`가 맡는다. `LIVE_ANCHOR`(인스턴스 설정의 캠프 첫날) 기준 경과일을 일정 길이로 나눈 나머지로 그날 일차를 정하고, 현재 시:분으로 진행 중/다음 순서를 뽑는다. 자정~아침 첫 순서(07:00) 사이엔 "취침"으로 조용히 채우고 다음 일정만 보여 준다.

카드는 현재 순서(시각·제목·장소)를 강조하고 아래에 다음 순서를 흐린 톤으로 얹는다. 각 일정에는 성격에 맞는 이동 링크(강의→관련 말씀 본문, 프로그램→자료실 고정 공지, 식사→식단표, 자유 묵상→말씀 탭, 찬양·쉬는 시간→플레이리스트)를 한 칩으로 달아 "지금 뭘 보면 되는지"를 한 번의 탭으로 잇는다. 이동 대상은 각 일정의 `link` 타입으로 정해진다. 톤은 공지 highlight 카드(2px 테두리)보다 약한 1px 테두리·차분한 배경으로 두어, 공지가 홈의 주인공 자리를 유지하도록 위계를 명확히 했다.

---

## 6. 말씀 탭 — 목록에서 본문 페이지로

말씀 본문은 처음엔 아코디언으로 제자리에서 펼쳤으나, 본문이 긴 장(누가복음 15·23장 등)에서는 목록이 밀려 탐색이 불편했다. **목록 → 개별 본문 페이지 이동** 구조로 바꿔, 강의를 고르면 전용 페이지에서 본문을 읽게 했다. 강의는 주제 강의·개회 폐회·GBS·아침 묵상으로 묶고, 각 항목에 안정적 id를 부여해 라이브 카드의 링크와 상세 라우트가 같은 키로 연결되게 했다. 본문은 한/EN 토글을 지원한다. 뒤로가기는 본문 → 말씀 탭 → 홈의 2단계 계층으로 복귀한다.

---

## 7. 참가자 프로필

익명 로그인 기반 온보딩에서 이름·목장·프로필 사진을 설정한다. 사진은 원형 크롭 UI로 얼굴을 맞춰 올리고, 정보 탭과 이후 화면에서 원형 아바타로 쓴다. 이름 라벨은 참가자에게 익숙한 "이름"으로 표기한다.

---

## 8. 앱 아이콘 · 스플래시

- **아이콘**: 초기 그린 테마에서 포스터의 블루 테마로 전환하며 재디자인했다. 흰 배경에 밝은 청색 `#67BDDB` 반짝임 심볼 하나만 두어, 작은 크기에서도 또렷하고 앱 전체 색과 홈 화면 아이콘의 인상을 일치시켰다.
- **스플래시**: 콜드 스타트마다 흰 배경 위에서 아이콘 → 로딩 막대 → 홈으로 이어진다. 1단계는 아이콘만 보여 브랜드를 각인하고, 2단계에서 아이콘이 작아지며 위로 올라가고 그 아래 막대가 브랜드 그라데이션으로 채워진 뒤 페이드아웃으로 홈에 넘어간다. 타이밍은 상수로 분리해 차분한 인상을 주도록 넉넉히 맞췄다.

---

## 9. 인앱 토스트

앱을 사용하는 동안엔 시스템 알림이 뜨지 않는 플랫폼 동작이 있어, 포그라운드에서 새 공지가 등록되면 화면 상단에 인앱 토스트로 알린다. 흰 카드에 1px 테두리와 부드러운 그림자, 좌측 원형 블루 종 아이콘으로 공지임을 알린다. 위에서 슬라이드 인해 4초 뒤 사라지고, 카드를 탭하면 상세로 이동, 위로 스와이프하면 닫힌다(살짝 떼면 탭, 충분히 끌면 닫기로 구분).

## 10. 공지 첨부 디자인

목록에서는 일관성을, 상세에서는 원본 충실도를 우선했다.

- 홈 카드는 우측에 정사각 썸네일을 두어 길이가 다른 공지도 같은 리듬으로 정렬하고, 이미지가 없으면 썸네일을 생략해 본문이 폭 전체를 쓴다. 파일 첨부는 첨부 칩(파일명 10자 말줄임)으로 한 줄을 유지한다.
- 자료실(고정 공지) 카드는 상단 공백을 없애고 제목과 상대시간을 한 줄에 나란히 올려 시선이 바로 제목에 닿게 했다.
- 상세 페이지는 이미지를 원본 비율 그대로(가로 꽉, 높이 자동) 보여 정보 전달을 우선하고 크롭하지 않는다. 파일은 좌측 정사각 PDF 썸네일 + 우측 파일명·다운로드 안내로 구성하고 탭 시 새 탭으로 연다.

## 11. 디자인과 기술의 교차점

### iOS PWA 상태바
- iOS는 `theme-color` 메타가 있으면 그 값으로 상태바를 단색으로 칠한다. 헤더의 수채화 그라데이션을 상태바까지 자연스럽게 잇기 위해 이 동작을 고려했다.
- `apple-mobile-web-app-status-bar-style`은 **홈 화면 추가 시점에 한 번만** 읽혀, 값을 바꾸면 재설치해야 적용된다.
- Android는 상태바가 OS가 칠하는 단색이라 그라데이션이 불가능하며, 매니페스트 `theme_color`를 그라데이션 상단 색과 맞추는 것이 최선이다.

### 하단 탭 고정
- `position: fixed`로 붙였더니 iOS PWA에서 `viewport-fit=cover`와 동적 뷰포트가 맞물려 위치가 불안정했다. 최상위를 `100dvh` 세로 flex로 두고 본문만 내부 스크롤하게 전환, 하단 탭은 flex 자식으로 맨 아래에 배치해 해결했다. 하단 안전영역은 `env(safe-area-inset-bottom)` 패딩으로 유지한다.

### 뒤로가기
- 어느 경로로 들어왔든 항상 같은 부모 화면으로 복귀하게 해 위치를 예측 가능하게 했다. 자식 화면은 홈으로, 상세는 목록으로 돌아간다. 홈에서는 뒤로가기로 앱이 종료되지 않게 머무르고, 방배정에서는 검색만 해제한다.

## 12. 디자인 시스템 요약

- **색**: 위 팔레트. 인스턴스 설정 → Tailwind → 전체 페이지 일괄 적용.
- **폰트**: Gowun Batang(한글·영문), 영문 표어는 세리프 이탤릭.
- **카드**: 흰 배경, 1px 테두리(`#D4E6EC`), 둥근 모서리. 강조 카드는 2px 테두리.
- **헤더**: 수채화 그라데이션 + eyebrow + 제목.
- **여백**: 모바일 가독성을 위해 충분한 상하 여백.

---

## 13. 향후 개선

- 작은 UI 텍스트(공지 본문·하단 탭)의 가독성을 폰에서 점검하고, 필요 시 해당 부분만 산세리프로 분리.
- 포스터 원본 수채화 이미지를 헤더 배경으로 사용해 질감을 더 살리기.
- 다크 모드 대응 검토.

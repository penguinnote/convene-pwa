# 디자인 기록 (DESIGN.md)

이 문서는 말씀캠프 PWA의 디자인 의사결정 과정과 디자인 시스템을 정리한 기록이다. UI를 어떤 기준으로 만들었고, 무엇을 시도했으며, 왜 최종안을 선택했는지를 담는다.

---

## 1. 디자인 목표와 원칙

- **모던 미니멀, 화이트 베이스**: 정보(공지, 일정, 말씀)가 우선이므로 장식을 절제하고 여백과 가독성을 중심에 둔다.
- **브랜드 일관성**: 참가자는 캠프 포스터를 먼저 접한 뒤 앱을 설치한다. 앱이 포스터와 같은 색과 무드를 공유해야 하나의 캠페인으로 인식된다.
- **모바일 우선**: 80~100명이 폰으로 사용하므로 모든 결정의 기준은 좁은 화면과 작은 글씨의 가독성이다.

---

## 2. 색상 결정 과정

### 1차: 바질 그린
초기에는 차분한 바질 그린을 포인트 색으로 화이트 베이스에 적용했다. 캠프와 자연의 분위기에는 맞았으나, 캠프의 실제 브랜드를 반영한 색은 아니었다.

### 2차: 포스터 톤으로 전환
캠프 포스터를 확인한 결과, 실제 정체성은 **맑은 하늘빛, 아쿠아 파스텔 + 흰 빛 + 금빛 반짝임**의 서정적인 수채화 톤이었다. 그린은 포스터와 이질적이어서, 브랜드 일관성을 위해 전체 색을 포스터 계열로 전환했다.

### 최종 팔레트
파스텔을 그대로 UI에 쓰면 대비가 부족해, 포인트는 한 단계 진한 톤을 사용했다.

| 용도 | 값 |
|---|---|
| 강조 / 활성 / 링크 | `#3F7D99` |
| 제목 및 공지 제목 | `#2F5E72` |
| 카드 배경 | `#F2F8FA` |
| 테두리 | `#D4E6EC` |
| 본문 | `#21343C` |
| 보조 텍스트 | `#52707D` |
| 흐린 텍스트 | `#8AA6B3` |

## 3. 타이포그래피 결정 과정

영문 표어 "Where are you?"를 세리프 이탤릭으로 두기로 하면서, 한글 폰트가 이와 어울리는지가 관건이었다. 다음 순서로 탐색했다.

1. **Pretendard / Spoqa Han Sans Neo** — 깔끔한 산세리프. 그러나 세리프 영문 표어와 결이 맞지 않았다.
2. **Nanum Myeongjo** — 한글과 영문을 명조로 통일했으나 전통적인 붓터치가 강해 올드하게 보였다.
3. **Hahmlet** — 모던 고대비 세리프. 우아하나 다소 각졌다.
4. **Gowun Batang (최종)** — 부드럽고 둥근 바탕체. 포스터의 포근한 무드와 가장 잘 맞고, 영문 세리프와도 자연스럽게 어울린다.

### 결정 사항
- 전체 폰트를 **Gowun Batang**으로 통일한다.
- 제목은 가는 무게(weight 400)로 두어 포스터의 가늘고 우아한 인상을 따른다.
- 제목과 공지 제목의 색은 `#2F5E72`로 통일한다.
- 영문 표어는 세리프 이탤릭으로 포스터의 포인트를 살린다.

## 4. 레이아웃 결정

- **수채화 헤더**: 홈과 각 페이지 상단에 CSS 그라데이션(라디얼 + 리니어)으로 수채화 질감을 표현하고, 금빛 반짝임을 절제해서 배치한다. 명도와 채도는 "진하되 밝게"를 기준으로 조정했다.
- **주제 말씀 정렬**: 가운데 정렬과 왼쪽 정렬을 비교한 결과, 2줄 본문은 왼쪽 정렬이 읽기 편하고 앱 전체의 좌측 정렬 리듬과 일관되어 **왼쪽 정렬**을 택했다. 가운데 정렬은 한 줄짜리 전시형 문구에 적합하다.
- **공지 UX**: 홈에는 최신 2개만 노출하고 우측 상단에 상대시간(just now / N분 전)을 표시한다. 공지 클릭 시 상세 페이지로, 섹션의 "전체 보기"로 전체 목록 페이지로 이동한다.
- **하단 탭 순서**: 홈 / 일정 / 말씀 / 방배정 / 사진. 사진은 라우트가 아니라 외부 Google Photos 공유 앨범으로 연결한다.

### 홈 개편 — 말씀 블록 제거 + 라이브 현재 순서 카드
초기 홈은 히어로(표어 "아담아, 네가 어디 있느냐?") 아래에 창세기 3:9 blockquote "주제 말씀" 블록을 두었으나, 히어로가 이미 같은 본문을 담고 있어 정보가 중복이었다. 말씀 블록을 제거해 공지 섹션을 히어로 바로 아래로 끌어올렸고, 앱을 열면 가장 중요한 공지가 첫 화면에 바로 보이게 했다.

홈 중앙에는 "지금 진행 중"을 보여주는 **라이브 현재 순서 카드**를 둔다. 처음에는 기기 시계로 현재 순서를 자동 계산하고 D-Day 카운트다운까지 붙였으나, 실제 캠프는 집회가 길어지거나 식사가 당겨지는 등 순서가 10~40분씩 수시로 밀리고 당겨진다. 시계 기반 자동 표시는 "지금"을 자주 틀리게 짚어 오히려 신뢰를 떨어뜨린다. 그래서 자동 계산과 D-Day를 걷어내고, **관리자가 `config/live` 포인터를 손으로 넘기는 수동 방식**으로 바꿨다. 관리자가 "이 순서 시작"을 누른 순간이 참가자 홈에 그대로 "지금 진행 중"으로 반영되고, 지연이 생기면 변동 메모("점심 30분 지연")로 사람이 직접 상황을 전한다. 정확성을 사람 손에 맡겨 신뢰를 지키는 선택이다.

카드는 현재 순서(시각·제목·장소)를 강조하고, 아래에 다음 순서를 흐린 톤으로, 그리고 순서 제목에서 판별한 관련 말씀/자료실 링크를 칩으로 얹는다. 톤은 공지 highlight 카드(2px 테두리)보다 약한 1px 테두리·차분한 배경으로 두어, 공지가 홈의 주인공 자리를 유지하도록 위계를 명확히 했다. active가 아닐 때는 카드를 완전히 숨겨 홈을 비워 둔다.

## 5. 앱 아이콘

초기 아이콘은 그린 테마를 따랐으나, 전체 색을 포스터의 블루 테마로 전환하면서 아이콘도 함께 재디자인했다.

- 흰 배경 위에 밝은 청색 `#67BDDB` 반짝임 심볼을 두어 포스터의 청색과 금빛 반짝임 모티프를 단순화했다.
- 그린 계열 흔적을 제거해 앱 전체 색과 홈 화면 아이콘의 인상을 일치시켰다.
- 단순한 심볼 하나로 작은 크기에서도 또렷하게 보이게 했다.

---

## 6. 스플래시 화면

앱을 열 때 빈 화면이 잠깐 보이는 대신, 브랜드를 각인시키는 진입 연출을 넣었다.

- 콜드 스타트마다 흰 배경 전체화면 위에서 아이콘 → 로딩 막대 → 홈 순서로 이어진다.
- 1단계는 아이콘만 보여 브랜드를 먼저 인지시키고, 2단계에서 아이콘이 작아지며 위로 올라가고 그 아래 로딩 막대가 브랜드 그라데이션으로 채워진다.
- 로딩 막대는 실제 진행률이 아니라 진입을 매끄럽게 잇는 연출 장치이며, 다 채워진 뒤 페이드아웃으로 홈에 자연스럽게 넘어간다.
- 타이밍은 상수로 분리해 조정할 수 있게 했다. 아이콘 노출과 막대 채움 시간을 넉넉히 두어 급하지 않고 차분한 인상을 주도록 맞췄다.

## 7. 인앱 토스트

앱을 사용하는 동안에는 시스템 알림이 뜨지 않는 플랫폼 동작이 있어, 포그라운드에서 새 공지가 등록되면 화면 상단에 인앱 토스트로 알린다. 시스템 알림 대신 화면 안에서 처리해 흐름을 끊지 않고 새 소식을 전한다.

- 디자인은 흰 카드에 1px 테두리와 부드러운 그림자를 두고, 좌측에 원형 블루 종 아이콘을 배치해 공지임을 한눈에 알린다. 본문 글꼴은 앱 전체와 같은 Gowun Batang으로 통일한다.
- 등장 시 위에서 아래로 슬라이드 인하고, 4초 뒤 자동으로 사라진다. 정보를 강요하지 않으면서도 놓치지 않도록 머무는 시간을 짧게 두었다.
- 닫는 동작은 두 가지다. 카드를 탭하면 해당 공지 상세로 이동하고, 위로 스와이프하면 닫힌다. 거의 움직이지 않고 떼면 탭으로, 충분히 위로 끌면 닫기로 구분해 의도가 섞이지 않게 했다.

## 8. 공지 첨부 디자인

공지에 이미지와 파일을 붙일 수 있게 되면서, 목록과 상세에서 첨부를 어떻게 보여줄지 기준을 정했다. 목록에서는 일관성을, 상세에서는 원본 충실도를 우선했다.

- 홈 카드는 우측에 정사각 썸네일을 두어 길이가 다른 공지도 같은 리듬으로 정렬된다. 썸네일 밑단을 본문 밑단에 맞춰 시선이 흐트러지지 않게 했다. 이미지가 없으면 썸네일을 생략하고 본문이 폭 전체를 쓴다.
- 파일 첨부는 홈 카드에서 첨부 칩으로 표시한다. 파일명은 확장자를 포함해 10자까지 보여주고 넘으면 말줄임해, 좁은 카드에서도 한 줄을 유지한다.
- 상세 페이지는 이미지를 원본 비율 그대로 보여준다. 가로를 꽉 채우되 높이를 자동으로 두어 잘리지 않게 했다. 공지 이미지는 정보 전달이 목적이므로 크롭하지 않는다.
- 상세의 파일은 좌측 정사각 PDF 썸네일 카드와 우측 파일명, 보기 및 다운로드 안내로 구성한다. 파일임을 한눈에 알리고 탭하면 새 탭으로 연다.

## 9. 디자인과 기술의 교차점에서 마주친 문제

### iOS PWA 상태바
- iOS는 `theme-color` 메타가 있으면 그 값으로 상태바를 단색으로 칠한다. 헤더의 수채화 그라데이션을 상태바까지 잇기 위해서는 이 동작을 고려해야 한다.
- `apple-mobile-web-app-status-bar-style`은 **홈 화면에 추가하는 시점에 한 번만 읽힌다.** 따라서 스타일을 바꾼 뒤에는 앱을 삭제하고 재설치해야 적용된다.
- Android는 상태바가 OS가 칠하는 단색 영역이라 그라데이션이 불가능하며, 매니페스트 `theme_color`를 그라데이션 상단 색과 맞춰 자연스럽게 잇는 것이 최선이다.

### 다이어그램 렌더링
- README의 아키텍처 다이어그램을 Mermaid로 작성했으나 GitHub에서 렌더링이 깨졌다. 라벨의 특수문자와 줄바꿈에 민감해 안정적이지 않았다.
- 어디서든 동일하게 보이도록 다이어그램을 **PNG 이미지로 교체**해 안정성을 확보했다.

### 하단 탭 고정
- 하단 탭을 `position: fixed`로 화면 맨 아래에 붙였으나, iOS PWA에서는 `viewport-fit=cover`와 동적 뷰포트가 맞물려 위치가 불안정했다. 스크롤이나 주소 표시줄 변화에 따라 탭이 밀리거나 떠 보이는 문제가 있었다.
- 최상위 컨테이너를 `100dvh` 높이의 세로 flex 레이아웃으로 두고, 본문 영역만 내부 스크롤하도록 전환했다. 하단 탭은 flex의 일반 자식으로 맨 아래에 배치해 고정 문제를 해결했다. 하단 안전영역은 `env(safe-area-inset-bottom)` 패딩으로 유지한다.

### 뒤로가기 동작
- 뒤로가기를 계층적으로 설계했다. 어느 경로로 들어왔든 항상 같은 부모 화면으로 복귀하게 해, 사용자가 위치를 예측할 수 있게 했다. 자식 화면은 홈으로, 공지 상세는 목록으로 돌아간다.
- 홈에서는 뒤로가기로 앱이 종료되지 않게 머무르고, 방배정에서는 뒤로가기로 검색만 해제해 화면을 벗어나지 않게 했다. 브라우저의 기본 뒤로가기 순서에 의존하지 않고 정보 구조에 맞춘 이동을 우선한 결정이다.

## 10. 디자인 시스템 요약

- **색**: 위 팔레트 표 참조. Tailwind 팔레트 한 곳에서 관리해 페이지 전체에 일괄 적용한다.
- **폰트**: Gowun Batang(한글과 영문), 영문 표어는 세리프 이탤릭.
- **카드**: 흰 배경, 1px 테두리(`#D4E6EC`), 둥근 모서리. 강조 카드는 2px 테두리.
- **헤더**: 수채화 그라데이션 배경 + eyebrow + 제목.
- **여백**: 모바일 가독성을 위해 충분한 상하 여백을 둔다.

## 11. 향후 개선

- 작은 UI 텍스트(공지 본문과 하단 탭)의 가독성을 폰에서 점검하고, 필요 시 해당 부분만 산세리프로 분리한다.
- 포스터 원본 수채화 이미지를 헤더 배경으로 사용해 질감을 더 살린다.
- 다크 모드 대응을 검토한다.
